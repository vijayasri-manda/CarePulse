# CarePulse — Backend API Service

This is the backend API service for **CarePulse**, built using Spring Boot. It provides a RESTful API for Patient Electronic Health Records (EHR), scheduling, billing, and authentication workflows.

---

## 🛠️ Technology Stack
* **Framework**: Spring Boot 3.x (Java 17)
* **Security**: Spring Security + Stateless JWT Authentication
* **ORM / Database**: Spring Data JPA, Hibernate, MySQL Driver
* **API Documentation**: Springdoc OpenAPI / Swagger UI
* **Mailing**: Spring Boot Starter Mail

---

## ⚙️ Environment Configuration

The application supports loading configuration variables either from a local `.env` file (processed dynamically via `dotenv-java`) or from standard environment variables (best for production/cloud platforms).

### Available Environment Variables

| Variable Name | Alternative (Spring Standard) | Description | Default / Example |
| :--- | :--- | :--- | :--- |
| `url` | `SPRING_DATASOURCE_URL` | JDBC URL for the MySQL connection | `jdbc:mysql://localhost:3306/hospitaldb` |
| `username` | `SPRING_DATASOURCE_USERNAME` | Database username | `root` |
| `password` | `SPRING_DATASOURCE_PASSWORD` | Database password | `your_password` |
| `CORS_ALLOWED_ORIGINS` | - | Comma-separated allowed frontend origins | `http://localhost:5173,http://localhost:3000` |
| `jwt.secret` | - | Strong signing key for JWT tokens (Min 256-bit) | *(Autogenerated safe fallback key)* |
| `PORT` | - | Port Spring Boot binds to | `8080` |

## 🚀 Local Development Setup

### 1. Prerequisites
- **Java Development Kit (JDK)**: Version 17 or higher
- **Maven**: 3.x or Maven Wrapper (included)
- **Database**: Active MySQL instance (local or cloud-hosted)

### 2. Properties Setup
Create a file named `.env` in the `backend/` directory:
```env
url=jdbc:mysql://localhost:3086/hospitaldb?useSSL=false&serverTimezone=UTC
username=YOUR_MYSQL_USERNAME
password=YOUR_MYSQL_PASSWORD
CORS_ALLOWED_ORIGINS=http://localhost:5173
jwt.secret=your_super_secret_minimum_256_bit_long_jwt_signing_key_here
```

### 3. Running Locally
Run the following Maven wrapper command to spin up the server:
```bash
# Windows
mvnw.cmd spring-boot:run

# Linux / macOS
./mvnw spring-boot:run
```
The API will run on `http://localhost:8080`. You can access the Interactive Swagger UI at:
`http://localhost:8080/swagger-ui/index.html`

## 📦 Deploying to Render

Render does not natively support Java services out of the box. However, we can deploy the backend using the root `Dockerfile` as a **Docker Web Service**.

### Step-by-Step Render Deployment Guide

1. **Create a New Web Service on Render**:
   - Log in to your [Render Dashboard](https://dashboard.render.com).
   - Click **New +** and select **Web Service**.
   - Connect your GitHub Repository containing this project.

2. **Configure Service Settings**:
   - **Name**: `carepulse-backend` (or your preferred name)
   - **Environment/Runtime**: Select **Docker**.
   - **Docker Context**: `/` (Leave as default root, as the Dockerfile resides in the root directory and copies files from `/backend`).
   - **Docker Path**: `Dockerfile` (Targeting the root Dockerfile).
   - **Instance Type**: Select your tier (Free or Starter).

3. **Configure Environment Variables**:
   Under the **Environment** tab, click **Add Environment Variable** and define:
   - `url` : *Your hosted MySQL Connection string* (e.g. Aiven, AWS RDS, PlanetScale).
   - `username` : *Database username*
   - `password` : *Database password*
   - `CORS_ALLOWED_ORIGINS` : *The URL of your frontend on Vercel* (e.g. `https://carepulse.vercel.app`).
   - `jwt.secret` : *A strong, custom 256-bit JWT secret string*.

4. **Deploy**:
   Click **Deploy Web Service**. Render will build the Docker container using Maven and spin up the Spring Boot REST API. It will automatically detect port `8080` (or the dynamic `PORT` assigned by Render) and make the backend live!
